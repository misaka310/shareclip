# ShareClip の使い方

## 起動

通常は、リポジトリ直下の次のbatをダブルクリックします。

```text
Start-ShareClip.bat
```

このbatが Node.js / npm の確認、初回 `npm install`、アプリ起動まで行います。

コマンドで起動したい場合は次です。

```bash
npm install
npm run dev
```

初回起動時に OCI 設定が未入力なら、自動で `設定` 画面を開きます。通常は JSON ファイルを手で書く必要はありません。

## 初回設定

1. `設定` 画面を開く
2. `Endpoint` / `Region` / `Bucket` / `Access Key ID` / `Secret Access Key` などを入力する
3. `設定を保存` を押す
4. `アップロード` に戻る

設定は Electron の user data に保存されます。`config/shareclip.config.local.json` を手で作る方法もありますが、これは自動化や開発者向けの任意手順です。

## アップロード

1. `アップロード` を開く
2. 期限を `1日` / `3日` / `7日` から選ぶ
3. ファイルをドラッグ&ドロップする、または `ファイルを選択` を押す
4. `アップロードしてリンク生成` を押す
5. 成功すると署名付き URL が表示され、クリップボードにもコピーされる
6. `共有リンク` の `コピー` で最新リンクを再コピーできる。コピー後はボタン表示が一時的に `コピーしました` に変わる

画面上の URL 表示は一部マスクされる場合があります。ダウンロード確認はシークレットウィンドウなど、認証を持たない環境で行います。

## 履歴

1. `履歴` を開く
2. 過去にアップロードしたファイル一覧を確認する
3. `コピー` で現在の URL を再コピーする。コピー後はボタン表示が一時的に `コピーしました` に変わる
4. `1日` / `3日` / `7日` で同じファイルの新しい署名付き URL を再発行してコピーする。コピー後は `コピー` ボタン側の表示も一時的に `コピーしました` に変わる
5. `削除` で OCI 上のファイルとローカル履歴を削除する

`1日` / `3日` / `7日` は、すでにコピー済みの URL の期限を後から変更するボタンではありません。新しい URL を作ってコピーするボタンです。古い URL は、その期限が切れるか、対象ファイルを削除するまで有効です。

## 設定

1. `設定` を開く
2. `接続準備OK` と `設定元` で現在状態を確認する
3. `Endpoint` / `Region` / `Bucket` / `Access Key ID` / `Secret Access Key` などを編集する
4. `設定を保存` を押す
5. 保存後は `アップロード` に戻って動作確認する

UI から保存した設定は Electron の user data にも保存されます。

## 配布用 exe を作る

配布用フォルダを作る場合は、次のbatをダブルクリックします。

```text
Build-ShareClip.bat
```

成功すると `release\ShareClip-win32-x64\ShareClip.exe` ができます。`ShareClip.exe` だけを抜き出さず、`ShareClip-win32-x64` フォルダごと使ってください。

## コマンドでの検証

変更時のコード確認は次を実行します。

```bash
npm run validate
npm run typecheck
npm test
npm run build
```

OCI 導通をコマンドだけで切り分けたい場合は、任意で `config/shareclip.config.local.json` を作ってから次を実行します。

```bash
npm run storage:check
```

`storage:check` は、実際に OCI へテストファイルをアップロードし、署名付き URL のダウンロード確認と後片付けまで行います。アプリの `設定` 画面に保存した値ではなく、ローカル設定ファイルを使います。
