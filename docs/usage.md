# ShareClip の使い方

## 起動

```bash
npm run dev
```

起動前に、次のローカル設定ファイルへ OCI 情報を入れておきます。

```text
config/shareclip.config.local.json
```

このファイルは Git に含めません。

## アップロード

1. `アップロード` を開く
2. 期限を `1日` / `3日` / `7日` から選ぶ
3. ファイルをドラッグ&ドロップする、または `ファイルを選択` を押す
4. `アップロードしてリンク生成` を押す
5. 成功すると署名付き URL が表示され、クリップボードにもコピーされる
6. `共有リンク` の `コピー` で最新リンクを再コピーできる

画面上の URL 表示は一部マスクされる場合があります。ダウンロード確認はシークレットウィンドウなど、認証を持たない環境で行います。

## 履歴

1. `履歴` を開く
2. 過去にアップロードしたファイル一覧を確認する
3. `コピー` で現在の URL を再コピーする
4. `1日` / `3日` / `7日` で同じファイルの新しい署名付き URL を再発行してコピーする
5. `削除` で OCI 上のファイルとローカル履歴を削除する

`1日` / `3日` / `7日` は、すでにコピー済みの URL の期限を後から変更するボタンではありません。新しい URL を作ってコピーするボタンです。古い URL は、その期限が切れるか、対象ファイルを削除するまで有効です。

## 設定

1. `設定` を開く
2. `接続準備OK` と `設定元` で現在状態を確認する
3. `Endpoint` / `Region` / `Bucket` / `Access Key ID` / `Secret Access Key` などを編集する
4. `設定を保存` を押す
5. 保存後は `アップロード` に戻って動作確認する

UI から保存した設定は Electron の user data にも保存されます。

## コマンドでの検証

普段の確認はこれで十分です。

```bash
npm run storage:check
```

変更時に一通り確認する場合は次を実行します。

```bash
npm run validate
npm run typecheck
npm test
npm run build
npm run storage:check
```

`storage:check` は、実際に OCI へテストファイルをアップロードし、署名付き URL のダウンロード確認と後片付けまで行います。
